Forbidden

What does this mean?
It means that the request contained valid data and was understood by the server, but the server is refusing action.

Why is this happening?
It could be due to one of a few things...

  • The user does not have the necessary permissions for a resource
  • A prohibited action has been attempted
  • The request provided authentication by answering the "WWW-Authenticate" header field challenge, but the server did not accept that authentication. The request should not be repeated.
